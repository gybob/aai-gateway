# AAI Gateway — 一个 MCP，统一所有 AI 代理

**安装一次 MCP 服务器和技能，即可跨所有 AI 代理共享。无需重启。无上下文膨胀。直接提问即可。**

## 问题所在

随着 MCP 生态系统的扩展，每个 AI 代理用户都会遇到同样的困境：

| 痛点           | 问题                                                                                                    |
| -------------- | ------------------------------------------------------------------------------------------------------- |
| **上下文膨胀** | 10 个 MCP 服务器 × 5 个工具 = 50 个完整的工具模式注入每个提示词，在模型开始思考之前就消耗了数千个 token |
| **重复配置**   | Claude Code、Codex、OpenCode — 需要手动同步三份相同的 MCP 服务器配置                                    |
| **需要重启**   | 添加新的 MCP？重启你的代理。每次都要。                                                                  |
| **找工具困难** | 搜索 GitHub、阅读 README、复制 JSON 配置、调试连接错误 — 所有这些都在你真正尝试使用工具之前就要完成     |

## 解决方案

AAI Gateway 位于 AI 代理和所有工具之间。一个 MCP 连接可以替代数十个。

|                | 无 AAI Gateway               | 有 AAI Gateway                                 |
| -------------- | ---------------------------- | ---------------------------------------------- |
| **上下文成本** | 每个提示词包含 50 个工具模式 | 10 个一行摘要（约 200 字符每个），详情按需加载 |
| **配置**       | 每个代理单独配置 MCP         | 导入一次，所有代理立即共享                     |
| **新工具**     | 安装后需要重启代理           | 热加载，立即可用                               |
| **发现**       | 手动搜索 + 复制配置          | `"帮我找一个文件系统 MCP"` → 秒级安装          |

## 快速开始（30 秒）

### 1. 将 AAI Gateway 添加到你的代理

**Claude Code**

```bash
claude mcp add --scope user --transport stdio aai-gateway -- npx -y aai-gateway
```

**Codex**

```bash
codex mcp add aai-gateway -- npx -y aai-gateway
```

**OpenCode** — 添加到 `~/.config/opencode/opencode.json`：

```json
{
  "mcp": {
    "aai-gateway": {
      "type": "local",
      "command": ["npx", "-y", "aai-gateway"],
      "enabled": true
    }
  }
}
```

### 2. 开始使用

直接和你的 AI 代理对话：

> "帮我搜索一个文件系统 MCP 并安装它"

> "导入这个 MCP：`npx -y @anthropic-ai/mcp-server-fetch`"

> "我安装了哪些工具？"

就这样。无需编辑配置文件，无需重启代理。

## 工作原理：两阶段披露

这是核心创新。AAI Gateway 使用**渐进式披露**，而不是将所有工具模式倾倒到提示词中：

```
┌─────────────────────────────────────────────────────────────────────┐
│  阶段 1 — 代理在工具列表中看到的内容                                  │
│                                                                     │
│  guide:filesystem    "读取/写入本地文件和目录"                      │ ~50 字符
│  guide:github        "管理 GitHub 仓库、议题和 PR"                   │ ~50 字符
│  guide:slack         "发送消息和管理 Slack 频道"                     │ ~50 字符
│  ... (每个应用一行，无参数模式)                                       │
├─────────────────────────────────────────────────────────────────────┤
│  阶段 2 — 代理决定使用文件系统                                       │
│                                                                     │
│  → 调用 guide:filesystem                                           │
│  ← 获取完整工具列表 + 参数模式 + 使用示例                            │
│  → 通过 aai:exec { app, tool, args } 执行                           │
└─────────────────────────────────────────────────────────────────────┘
```

**计算**：10 个 MCP 服务器，每个 5 个工具 = 传统设置中 **50 个完整模式**。使用 AAI Gateway = **10 个简短摘要** + 仅在需要时加载详情。上下文节省 **90%+**。

## 核心特性

### 自然语言搜索与安装

描述你的需求，AAI Gateway 帮你找到。它搜索可信来源（官方 MCP 注册表、精选列表），展示选项，并处理整个导入流程。

> "找一个用于数据库查询的 MCP" → 搜索 → 选择 → 导入 → 就绪

### 导入任何 MCP 服务器

粘贴任何标准 MCP 配置，让你的代理通过 AAI Gateway 导入。

**stdio MCP：**

```json
{ "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"] }
```

**远程（Streamable HTTP）：**

```json
{ "url": "https://example.com/mcp" }
```

**远程（SSE）：**

```json
{ "url": "https://example.com/sse", "transport": "sse" }
```

### 导入技能

导入本地或远程技能包。AAI Gateway 将它们复制到托管存储并作为工具暴露。后续更新：自动从远程源更新技能 — 解决技能作者无法向用户推送更新的痛点。

```json
{ "path": "/path/to/my-skill" }
```

### 代理互操作性（ACP）

AAI Gateway 自动检测已安装的代理工具（Claude Code、Codex、OpenCode）并将其作为可控制的应用暴露。这意味着：

- 使用**一个代理编排另一个** — 例如，让 Claude Code 写代码，同时你从另一个工具审查
- **远程工作** — 在外出时用手机指挥你的编码代理

### 代理级别控制

一次导入为所有代理服务，但你可以微调可见性：

- `enableApp` / `disableApp` — 按代理切换工具
- `removeApp` — 完全卸载
- `listAllAaiApps` — 查看所有已注册的内容

### 自动发现

应用可以通过在已知位置放置 `aai.json` 描述符来注册自己。AAI Gateway 自动扫描这些位置：

- **macOS**：`<App>.app/Contents/Resources/aai.json`
- **Linux**：`/usr/share`、`/usr/local/share`、`~/.local/share`
- **Windows**：`Program Files`、`%LOCALAPPDATA%`

无需手动导入 — 安装应用后，AAI Gateway 会自动找到它。

## 内置工具

| 工具              | 描述                                                |
| ----------------- | --------------------------------------------------- |
| `listAllAaiApps`  | 列出 AAI Gateway 管理的所有应用                     |
| `enableApp`       | 为当前代理启用某个应用                              |
| `disableApp`      | 为当前代理禁用某个应用                              |
| `removeApp`       | 从系统中移除某个应用                                |
| `aai:exec`        | 执行托管应用中的特定工具（`app` + `tool` + `args`） |
| `mcp:import`      | 导入 MCP 服务器                                     |
| `skill:import`    | 导入技能包                                          |
| `skill:create`    | 创建新技能                                          |
| `search:discover` | 用自然语言搜索新工具或技能                          |

另外还有每个已导入应用的 **`guide:<app-id>`** 工具 — 无参数，调用时返回完整操作指南。

## 使用场景

### "我有 15 个 MCP，上下文快溢出了"

AAI Gateway 的两阶段披露将上下文 token 使用量减少 90%+。你的代理看到简短摘要，而不是 15 个完整的工具模式。

### "我同时用 Claude Code 和 OpenCode"

通过 AAI Gateway 导入一次。两个代理立即看到相同的工具。明天添加 Codex — 它也会获得，零额外配置。

### "我想喝茶的时候写代码"

设置 ACP 代理。使用任何代理（甚至手机上的）来指示 Claude Code 或 Codex 在你的工作站上编写、测试和提交代码。

### "我不知道该用哪个 MCP"

只需描述你的需求：`"我需要查询 PostgreSQL 的工具"`。AAI Gateway 搜索可信注册表并处理整个安装过程。

## 应用开发者：AAI 描述符

想让你的应用被 AAI Gateway 自动发现？添加一个 `aai.json` 描述符：

```json
{
  "schemaVersion": "2.0",
  "version": "1.0.0",
  "app": {
    "name": { "default": "My App" }
  },
  "access": {
    "protocol": "mcp",
    "config": {
      "command": "my-app-mcp",
      "args": ["--stdio"]
    }
  },
  "exposure": {
    "summary": "当用户想要用 My App 做 X 时使用。"
  }
}
```

支持的协议：`mcp`、`skill`、`acp-agent`、`cli`

将描述符放置在[自动发现位置](#自动发现)之一，AAI Gateway 会自动获取它。

想将你的描述符默认打包到 AAI Gateway？[提交 PR](../../pulls) — 参考 `src/discovery/descriptors/` 中的示例。
